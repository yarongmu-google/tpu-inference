# Tuning v2 migration plan

This branch (`tune`) rewrites the kernel-tune + service-sweep + benchmark pipeline to match the design in `docs/tuning_architecture.md`. The rewrite lands as a sequence of independently-reviewable commits; each commit ships with tests and runs standalone. v1 stays parked under `tools/kernel/tuner/v1/`, `tools/benchmark/sweep*.py`, etc., during the migration so the rpa3_2 branch keeps working.

## Design references

- **Architecture**: `docs/tuning_architecture.md` (the load-bearing spec; this branch implements it).
- **Variable classification**: `tuning_architecture.md` §2 (six categories: workload key / measurement var / kernel-tuned / kernel-derived / service-tuned / service-derived).
- **Kernel ↔ Service symmetry**: §3.
- **Storage layout**: §4-5 (append-only JSONL, per-SHA partitioned, TTL=2).
- **Two lookup directions**: §7 (top-down deploy, bottom-up impact-analysis).
- **Migration sequence**: §12 items 1-8.

## File layout (generated by v2)

All paths are git-tracked. 🅗 = hand-edited; 🅖 = generated.

### Per workload
```
tools/benchmark/cases/<topo>/<model>/
    <workload>.workload                              🅗  bash file: model + measurement vars
    <workload>.kernel_axes.json                      🅗  optional per-workload kernel search-space override
    <workload>.service_axes.json                     🅗  optional per-workload service search-space override

    <workload>.kernel.raw/                           🅖  by tune_kernel.sh — long, periodic commit
        <kernel_sha>.jsonl                                one file per kernel-source SHA, append-only, TTL=2
    <workload>.kernel                                🅖  by project_kernel.sh — short, commit-once
                                                          winners per tuning_key

    <workload>.service.raw/                          🅖  by sweep_service.sh — long, periodic commit
        <service_sha>.jsonl                               one file per (vllm_sha, runner_sha), append-only, TTL=2
    <workload>.service                               🅖  by project_service.sh — short, commit-once
                                                          winners per objective
```

### Per model (accumulator)
```
tools/benchmark/cases/<topo>/<model>/
    production.kernel                                🅖  by aggregate.sh — short, commit-once
                                                          union of all <workload>.kernel under this model
    production.service                               🅖  by aggregate.sh — short, commit-once
                                                          union of all <workload>.service under this model
```

No cross-model global accumulator. Per-model is the unit.

## Lifecycle per generated file

| Path | Producer | Consumer | When committed |
|---|---|---|---|
| `<workload>.workload` | Operator | All drivers | On edit |
| `<workload>.kernel_axes.json` | Operator (optional) | Tune | On edit |
| `<workload>.service_axes.json` | Operator (optional) | Sweep | On edit |
| `<workload>.kernel.raw/<sha>.jsonl` | tune_kernel.sh | project_kernel.sh | Periodic during run + once at end |
| `<workload>.kernel` | project_kernel.sh | lookup + auto-link + aggregate | Once on completion |
| `<workload>.service.raw/<sha>.jsonl` | sweep_service.sh | project_service.sh | Periodic during run + once at end |
| `<workload>.service` | project_service.sh | lookup + aggregate | Once on completion |
| `production.kernel` | aggregate.sh | impact-analysis tooling | Once on completion |
| `production.service` | aggregate.sh | impact-analysis tooling | Once on completion |

## Invariants enforced

- **`mns × kernel_K == LONG_PREFILL_TOKEN_THRESHOLD`** (server-side, runner init).
- **`BLOCK_SIZE == kernel-tuned page_size`** (server-side, runner init).
- **`RPA_KERNEL_K == kernel-tuned chunk_prefill_size`** (server-side, runner init).
- **`<workload>.kernel` row's `code_revision` matches the kernel_sha that produced the `.raw/<sha>.jsonl`** (projection-time check).
- **`<workload>.service` row's `kernel_pin_keys` matches an existing `<workload>.kernel` row** (projection-time check; missing kernel → reject the service combo).

## Out of scope for this branch

- Distributed-tuning sharding across multiple VMs (per-pin-key range-partition).
- A web UI for impact-analysis browsing.
- Per-objective `.service` winner selection beyond `throughput_max`, `ttft_min`, `p99_min`.
- The .service.raw entries beyond what `vllm bench serve` already produces.
